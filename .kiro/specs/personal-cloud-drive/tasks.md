# 实现计划：个人网盘系统（MVP）

## 概述

基于 Monorepo（pnpm workspace）组织，分为 `packages/client`（Electron 桌面客户端）、`packages/server`（NestJS API 服务 + Worker）、`packages/shared`（共享类型与常量）三个工作区。实现顺序遵循「基础设施 → 后端核心 → 客户端核心 → 集成联调」的渐进路径，每个阶段结束设置检查点。

## 任务

- [x] 1. 工程初始化与基础设施搭建
  - 初始化 pnpm workspace Monorepo 结构（client / server / shared）
  - 配置 TypeScript、ESLint、Prettier、Vitest 等工程化工具
  - 搭建 Electron Forge + React + Webpack 客户端脚手架
  - 搭建 NestJS 后端项目骨架（含 Prisma、BullMQ、ioredis、pino 日志）
  - 编写 Docker Compose 配置（PostgreSQL、Redis、MinIO）
  - 定义 shared 包中的公共类型、常量与边界值
  - _需求覆盖：全局工程基础_

- [ ] 2. 数据库设计与 ORM 层
  - 编写 Prisma Schema（user、user_profile、user_quota、file_object、file_entry、folder_entry、upload_session、upload_part、download_session、async_task）
  - 创建数据库迁移脚本
  - 编写关键索引（文件列表复合索引、搜索索引、MD5 唯一索引）
  - _需求覆盖：需求 1-10 数据层基础_

- [ ] 3. 用户认证模块（后端 + 客户端）

  - 客户端实现登录/注册页面 UI
  - 客户端实现 Token 安全存储（keytar）、自动登录、记住密码
  - 客户端实现 Axios 拦截器（自动附加 Token、过期自动刷新）
